The Open- Suite

OpenResponses

A "Super Beta" native iOS client for the OpenAI Responses API, engineered for power users to explore the bleeding edge of agentic AI. Features an "Ultra-strict" Computer Use mode that disables app-side helpers for pure model-driven browser automation.

  • Advanced Model Support: Day-one support for GPT-5, o1 (Reasoning), and the dedicated computer-use-preview model.
  • Agentic Architecture: Implements a full "Perceive-Reason-Act" loop with 100% action coverage for the computer tool (clicking, typing, scrolling).
  • Model Context Protocol (MCP): Built-in registry for MCP Connectors including Dropbox, Gmail, and Google Calendar, plus support for custom MCP servers.
  • Enterprise Features: Includes Prompt Caching for low-latency workflows, Structured Outputs (JSON Schema/Pydantic), and Code Interpreter (Python sandbox).
  • Observability: Real-time streaming of "Thinking" states, tool calls, and token usage with semantic server-sent events.

OpenIntelligence

A forward-looking, privacy-first RAG system architected for iOS 26+. It leverages Apple Intelligence (Foundation Models) and Private Cloud Compute (PCC) to deliver zero-retention AI processing.

  • Hybrid Search Engine: A sophisticated retrieval pipeline combining BM25 (keyword) and Vector Search (Cosine Similarity) using Reciprocal Rank Fusion (RRF).
  • Protocol-Oriented Design: Modular LLMService architecture allowing hot-swapping between Apple Foundation Models, OpenAI, and on-device Core ML (GGUF) backends.
  • On-Device Processing: Uses NLEmbedding for 512-dim vector generation and PDFKit/Vision for local OCR and semantic chunking (400w windows).
  • Performance: Achieves <100ms embedding generation and <50ms vector search for 1K chunks using highly optimized Swift concurrency.
Swift 6 Actors Private Cloud Compute Core ML Hybrid Search
GitHub

OpenCone

A production-grade on-device RAG application that transforms personal document collections into a secure, searchable knowledge base using Pinecone and OpenAI.

  • MVVM-S Architecture: Built on a strict Model-View-ViewModel-Service pattern with dependency injection for maximum testability and separation of concerns.
  • Async Pipeline: A complex, non-blocking ingestion pipeline using Swift's async/await to handle PDF text extraction, MIME-type aware chunking, and vector upserts.
  • Reactive Data Flow: Extensive use of Combine and @Published properties for real-time UI updates and a decoupled, singleton-based logging system.
  • Security & Persistence: Utilizes Security-Scoped Bookmarks for persistent file access and Keychain for secure API key storage.
SwiftUI Pinecone Combine Async/Await Security-Scoped Bookmarks
App Store GitHub

OpenAssistant

A comprehensive, native SwiftUI client for the OpenAI Assistants API, designed to manage the full lifecycle of AI assistants, threads, and runs.

  • Assistant Lifecycle: Full CRUD capabilities for Assistants, enabling precise configuration of instructions, models (GPT-4o), and tools (Code Interpreter, File Search).
  • Vector Store Management: Dedicated UI for managing Vector Stores, including file uploads, association with assistants, and chunking strategy configuration.
  • Robust Networking: A specialized APIService layer handling multipart file uploads, error recovery, and server-sent events for streaming responses.
  • State Management: Leverages ObservableObject and NotificationCenter for decoupled communication across the app's MVVM architecture.
